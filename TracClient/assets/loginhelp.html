Specify TRAC server

For more information on starting TracClient form an URL see this page.

On this screen can be specified which TRAC server should be used.
You can save the information on your device. Be aware that passwords are stored in plain text, but on non rooted devices normally not available to other apps.
If saved information is available a dropdown box is presented to select one of the stored profiles.

If login information is required then it can also be entered.

Use the Verify button to check if the app can connect to the server. With OK the server information is stored.

Normally the URL to fill in is TRAC-addrres appended with /rpc if no login information is neededand /login/rpc if this is needed.
Some example URLs:

  • TracClient RO: http://van-loon.xs4all.nl/TracClient/rpc
    No need for username and password
  • TracClient RW: https://van-loon.xs4all.nl/TracClient/login/rpc
    A valid Username and password are needed and for this site the Selfsigned certifcate checkbox must be set.

Through the menu the saved profiles can be exported to the SD card. In that case the file TracClient.XML will be written in the directory TracClient. The import function again loads the information into the database, old information is discarded. Be aware that also the passwords will be exported.

The sort order is set to priority&modified&desc=1.
The filter is set to max=500&status!=closed.

SSL considerations

Self Signed Certificates

If you get the error No peer certificate or Trust anchor for certification path not found, select Accept Self Signed certificates; this effectively disables the SSL protection.

ServerName Indication

At the moment the app does not support ServerName Indiaction (SNI), this means that on systems with several hostnames with te same IP-address the SSL verification can go wrong. This will be recognized and you will be asked to ignore the hostname verification error. This also breaks the protection offered by SSL but lets you use the site. I am looking for a safer solution.
