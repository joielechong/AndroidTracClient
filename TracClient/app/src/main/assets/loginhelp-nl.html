Specificeer TRAC server

Voor meer informatie over het starten van TracClient vanuit een URL zie deze pagina.

Met dit scherm kan worden opgegeven welke TRAC server gebruikt dient te worden.
Je kunt de informatie op het apparaat opslaan. Wees ervan bewust dat wachtwoorden worden als platte tekst opgelsagen maar op niet gerootte apparaten is die normaliter niet beschikbaar zijn voor andere apps.
Als opgeslagen informatie beschikbaar is dan wordt een dropdown box getoond om een van de opgeslagen profielen te selecteren.

Indien login informatie vereist is dan kan deze ook ingevoerd worden.

Gebruik de Verifieer knop om te controleren of de app verbinding kan maken met de server. Met OK wordt de server informatie opgeslagen.

Normaal gesproken is de in te vullen URL het adres van de TRAC site aangevuld met /rpc als er geen login informatie nodig en /login/rpc indien deze wel vereist is.
Een paar voorbeeld URLs:

  • TracClient RO: http://van-loon.xs4all.nl/TracClient/rpc
    Geen gebruikersnaam en wachtwoord nodig
  • TracClient RW: https://van-loon.xs4all.nl/TracClient/login/rpc
    Een geldige gebruikersnaam en wachtwoord zijn nodig ook moet voor deze site het certificaat vinkje gezet worden.

Via het menu kunnen de opgeslagen profielen worden gexporteerd naar de SD-kaart. Dan wordt het file TracClient.XML geschreven in de TracClient folder. Met de importeer functie kan dit file weer worden ingelezen, de oude informatie wordt overschreven. Let op ook de wachtwoorden leesbaar worden geexporteerd.

De sorteervolgorde wordt op priority&modified&desc=1 gezet.
Het filter wordt op max=500&status!=closed gezet.

SSL overwegingen

Self Signed Certificaten

Als je de fout No peer certificate of Trust anchor for certification path not found krijgt, selecteer dan Accepteer Self Signed certificaten. In dit geval wordt de SSL bescherming omzeild.

ServerName Indication

Op dit moment ondersteunt de app geen Servername Indiaction (SNI). Dit betekent dat op systemen met meerdere hostnamen op hetzeklfde IP adres de SSL-verificatie fout kan gaan. Dit zal worden herkend en je wordt gevraagd om de hostname verificatie fout negeren. Dit breekt ook de bescherming door SSL, maar kunt u de site te gebruiken.
